# 📖 ГАЙДЛАЙНЫ: Как создавать новых агентов

## 🚀 БЫСТРЫЙ СТАРТ

### Способ 1: Автоматический скрипт (РЕКОМЕНДУЕТСЯ)

```bash
bash ./create-agent.sh "analyzer" "Анализирует качество кода и предлагает улучшения"
```

Скрипт автоматически:
1. Создаст папку `.claude/agents/analyzer/`
2. Скопирует `AGENT_TEMPLATE.md` → `analyzer.md`
3. Заполнит name, description
4. Добавит все наследуемые навыки из `base-config.yml`
5. Создаст `MEMORY.md` для сохранения результатов

### Способ 2: Ручной (если нужны кастомные настройки)

1. Скопируй `.claude/agents/AGENT_TEMPLATE.md`
2. Переименуй в `.claude/agents/[name]/[name].md`
3. Заполни:
   - `name:`
   - `description:`
   - `[СПЕЦИФИЧНЫЕ ИНСТРУКЦИИ]`
4. Всё остальное наследуется автоматически!

## 🎯 КАК ЗАПОЛНИТЬ FRONTMATTER

```yaml
---
name: my-analyzer          # [required] имя агента
description: Анализирует код и предлагает улучшения  # [required]
tools: Read, Grep, Glob, Bash, WebFetch, WebSearch   # [inherited]
model: haiku               # [inherited]
memory: project            # [inherited]
effort: high               # [required] low/medium/high
maxTurns: 10              # [inherited]
config:
  mode: budget             # [inherited] budget или production
  search: duckduckgo       # [inherited] всегда бесплатный
  cost: "$0.00"            # [inherited] бюджетный режим
---
```

**Легенда:**
- `[required]` - ты обязан заполнить
- `[inherited]` - наследуется из base-config.yml (можешь переопределить)

## 🔄 ВЗАИМОДЕЙСТВИЕ АГЕНТОВ

```markdown
## Входящие данные
← RESEARCH.md (от researcher)
← BRAINSTORM.md (от brainstormer)

Код:
Read: .claude/agents/RESEARCH.md
```

### Писать output для следующего агента

```markdown
## Исходящие данные
→ CRITIQUE.md (для critic агента)
→ MEMORY.md (для истории)

В конце ответа ты создаёшь:
- CRITIQUE.md с твоими результатами
- Обновляешь MEMORY.md
```

## 🎁 ЧТО ПОЛУЧИТ НОВЫЙ АГЕНТ АВТОМАТИЧЕСКИ

Когда создаёшь агента через скрипт или шаблон, он получит:

✅ **Инструменты:**
- Read, Grep, Glob (локальные файлы)
- WebFetch, WebSearch (интернет - через DuckDuckGo)
- Bash (выполнение команд)

✅ **LLM:**
- Ollama mistral (локально, $0.00)
- Или neural-chat если нужна логика

✅ **Фичи:**
- Бесплатный поиск (DuckDuckGo)
- Учёт стоимости токенов
- Взаимодействие с другими агентами
- Сохранение в MEMORY.md

✅ **Будущие фичи:**
- Все, что добавим в `base-config.yml → features`
- Автоматически наследуется БЕЗ обновления конфига!

## 🔮 КОГДА ДОБАВИМ НОВУЮ ФИЧУ

### Когда захочешь добавить новую возможность:

1. **Добавь в `base-config.yml`:**
```yaml
features:
  future_v2:
    - vector_search        # ← Новая фича
```

2. **Обновляется в `_agent-defaults.md`:**
```markdown
## 🚀 НОВЫЕ ФИЧИ
- vector_search ← все агенты получат это!
```

3. **Все агенты автоматически получат новую фичу!**
```
Не нужно:
❌ Обновлять каждого агента
❌ Переписывать конфиги
❌ Пересоздавать агентов

Просто:
✅ Добавь в base-config.yml
✅ Все получат автоматически!
```

## 🚀 БЫСТРЫЙ СКРИПТ: create-agent.sh

```bash
#!/bin/bash
# Использование: bash create-agent.sh "agent-name" "Description"

NAME=$1
DESCRIPTION=$2
DIR=".claude/agents/$NAME"

mkdir -p "$DIR"

# Скопируй шаблон
cp .claude/agents/AGENT_TEMPLATE.md "$DIR/$NAME.md"

# Заполни placeholder'ы
sed -i "s/\[AGENT_NAME\]/$NAME/g" "$DIR/$NAME.md"
sed -i "s/\[КРАТКОЕ ОПИСАНИЕ.*\]/$DESCRIPTION/g" "$DIR/$NAME.md"

# Создай MEMORY.md
cat > "$DIR/MEMORY.md" << EOF
# 📝 Memory: $NAME

Статус: Новый агент
Создан: $(date)

## История запусков
[будет заполняться автоматически]
EOF

echo "✅ Агент '$NAME' создан!"
echo "Редактируй: $DIR/$NAME.md"
```

## 📖 ДОКУМЕНТАЦИЯ

- **AGENT_TEMPLATE.md** — шаблон для копирования
- **base-config.yml** — глобальные настройки (наследуются все)
- **_agent-defaults.md** — стандартные инструкции (наследуются все)
- **GUIDELINES.md** — этот файл
- **AGENT_PIPELINE.md** — взаимодействие агентов

## 💬 ЧАСТЫЕ ВОПРОСЫ

**Q: Нужно ли обновлять все агенты когда добавляю новую фичу?**
A: НЕТ! Просто добавь в `base-config.yml → features` и все получат её автоматически.

**Q: Как новому агенту получить все навыки старых?**
A: Все наследуются из `base-config.yml` + `_agent-defaults.md` автоматически.

**Q: Могу ли я переопределить наследуемую настройку?**
A: ДА! Напиши её в своём frontmatter и она переопределит наследуемое значение.

**Q: Где сохранять результаты нового агента?**
A: В MEMORY.md в папке агента + специальном файле если нужно (CRITIQUE.md, DECISION.md и т.д.)
