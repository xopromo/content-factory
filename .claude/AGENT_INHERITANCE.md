# 🧬 Архитектура наследования агентов

## Проблема

❌ **Старый подход:**
```
Создаём agent-1 с best practices
  → Создаём agent-2, забываем одну фичу
  → Создаём agent-3, снова переписываем всё
  → Добавляем новую фичу → надо обновить ВСЕ агентов
  → Хаос и дублирование 😤
```

## Решение: НАСЛЕДОВАНИЕ

✅ **Новый подход:**
```
base-config.yml (глобальные настройки)
    ↓ наследуют ВСЕ агенты
_agent-defaults.md (стандартные инструкции)
    ↓ наследуют ВСЕ агенты
[new-agent].md (только специфичное!)
    ↓
Агент получает ВСЕ best practices + новые фичи автоматически!
```

## 🎯 Трёхслойная архитектура

```
LAYER 1: base-config.yml
┌─────────────────────────────────────────────────────┐
│ Глобальные настройки (для ВСЕХ агентов):            │
│ - Режимы: budget (Ollama) vs production (Claude)    │
│ - LLM: mistral, neural-chat, codellama              │
│ - Поиск: DuckDuckGo (бесплатно)                     │
│ - Features: нынешние + будущие                      │
│ - Стоимость: $0.00 (budget) или $0.35 (prod)       │
└─────────────────────────────────────────────────────┘
           ↓ наследует КАЖДЫЙ НОВЫЙ АГЕНТ

LAYER 2: _agent-defaults.md
┌─────────────────────────────────────────────────────┐
│ Стандартные инструкции (для ВСЕХ агентов):          │
│ - Язык: простой русский, без жаргона                │
│ - Инструменты: Read, Grep, Glob, WebFetch и т.д.   │
│ - Взаимодействие: как читать output других          │
│ - Требования: стоимость в конце, источники          │
│ - Примеры правильных ответов                        │
└─────────────────────────────────────────────────────┘
           ↓ наследует КАЖДЫЙ НОВЫЙ АГЕНТ

LAYER 3: [agent-name].md (СПЕЦИФИЧНОЕ ТОЛЬКО!)
┌─────────────────────────────────────────────────────┐
│ Уникальное для каждого агента:                      │
│ - Название и описание                               │
│ - Специфичные инструкции (шаги 1-2-3)              │
│ - Входящие/исходящие данные                         │
│ - Специальный формат вывода                         │
│                                                      │
│ ВСЁ ОСТАЛЬНОЕ = НАСЛЕДУЕТСЯ АВТОМАТИЧЕСКИ! ✅       │
└─────────────────────────────────────────────────────┘
```

## 🔄 Как добавить НОВУЮ фичу (для всех агентов сразу)

### Старый способ (было):
```
1. Придумаешь новую фичу
2. Обновишь agent-1.md
3. Обновишь agent-2.md
4. Обновишь agent-3.md
5. Обновишь agent-4.md
6. Забудешь что-нибудь 😤
```

### Новый способ (теперь):
```
1. Открыл .claude/agents/base-config.yml
2. Добавил в features.future_v2:
   ```yaml
   future_v2:
     - vector_search          # ← НОВАЯ ФИЧА!
     - local_rag
     - file_caching
   ```
3. ГОТОВО! ✅
4. Все агенты получают новую фичу автоматически!
5. Нет нужды обновлять каждого агента
6. Нет копирования/дублирования
```

## ✨ ПРИМЕРЫ НАСЛЕДОВАНИЯ

### Пример 1: Инструменты

```
base-config.yml:
  tools:
    - Read, Grep, Glob
    - WebFetch, WebSearch

researcher.md:
  [не указывает инструменты]

Результат:
✅ researcher получает ВСЕ инструменты
```

### Пример 2: Новая фича добавляется

```
БЫЛО (v1.0):
features:
  version_1:
    - offline_ollama
    - free_duckduckgo_search

СТАЛО (v1.1):
features:
  version_1:
    - offline_ollama
    - free_duckduckgo_search
  future_v2:
    - vector_search          # ← НОВАЯ!

РЕЗУЛЬТАТ:
✅ Все 7 агентов автоматически получат vector_search
❌ Не нужно обновлять ни одного агента
❌ Не нужно копировать/дублировать
```

### Пример 3: Переопределение настройки

```
base-config.yml говорит:
  model: haiku
  effort: medium

analyzer.md говорит:
  model: haiku
  effort: high          # ← переопределил!

РЕЗУЛЬТАТ:
✅ analyzer использует effort: high
✅ Все остальное наследует из base-config.yml
```

## 🎁 ЧТО ЭКОНОМИТ НАСЛЕДОВАНИЕ

### Когда создаёшь агента:

❌ **БЕЗ наследования (было):**
- 300+ строк конфига
- Копируй инструменты для каждого
- Копируй требования к output
- Копируй примеры ответов
- Дублирование = ошибки

✅ **С наследованием (теперь):**
- 50-100 строк специфичного
- Остальное наследуется автоматически
- Одно место для обновления фич
- Нет дублирования
- Централизованное управление

### Когда добавляешь новую фичу:

❌ **БЕЗ наследования (было):**
- Обновить 7 агентов: 7 × 10 минут = 70 минут
- Забыть одного = ошибка
- Много копирования

✅ **С наследованием (теперь):**
- Обновить 1 файл: 2 минуты
- Все получат автоматически
- Нет ошибок
